# Methodology

Each note should be short, sourced and defensively useful.

## Default structure

1. **Why this matters** — the reason the topic belongs in this portfolio.
2. **Executive summary** — the key points in plain language.
3. **Observed facts** — what is supported by sources.
4. **Analysis** — what the facts suggest, with uncertainty kept visible.
5. **Defensive relevance** — CTI, SOC, threat hunting, AppSec or AI Security lessons.
6. **Limits and caveats** — what is unknown, disputed or weakly sourced.
7. **Sources** — primary sources first, then strong secondary sources.

## Analytical rules

- Separate fact, inference and opinion.
- Prefer cautious language when attribution is uncertain.
- Track confidence levels explicitly.
- Avoid hype and cyber folklore.
- Explain why a case matters defensively.
