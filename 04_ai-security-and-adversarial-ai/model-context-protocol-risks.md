# Model Context Protocol risks

## Status

Structured stub — to be expanded with sources.

## Why this topic matters

MCP (Model Context Protocol) is relevant because it connects AI systems to tools, data sources and workflows. This can expand the attack surface for prompt injection, malicious tools, data exposure and agentic misuse.

## Analytical angle

- Tool trust and permission boundaries.
- Malicious or compromised connectors.
- Indirect prompt injection through retrieved context.
- Defensive logging and monitoring.
- Difference between architectural risk, experimental proof and field evidence.

## Defensive relevance

This topic is relevant for SOC, AppSec, AI Security and threat hunting teams preparing for agentic AI environments.
