# Prompt injection

**Type:** AI Security / LLM application risk / agentic systems

## Why it is on my radar

Prompt injection is one of the first AI Security topics that made me feel there was a real bridge between my interest in cybersecurity and my interest in AI. I encountered it through OWASP, AI Security discussions, agent security research and my own curiosity around LLMs.

## What stuck with me

What interests me is that the attack surface is semantic. The malicious input can be natural language, hidden instructions, retrieved context, a web page, an email, a document or a tool description.

It becomes much more serious when the model is not just chatting, but connected to tools, files, APIs or internal data.

## What I take from it

Prompt injection helped me understand that AI Security is not only about “making the model refuse bad requests”. The real question is what the system around the model can access and do.

For CTI (Cyber Threat Intelligence), it is also an emerging abuse pattern to watch carefully: what is theoretical, what is proven in labs, and what is actually seen in the field.

## Current limit

I understand the concept well, but I still need more hands-on practice testing real LLM applications and agent workflows.

## Resources I actually used

OWASP LLM Top 10, NCSC material, AI Security research discussions, content around indirect prompt injection, agents and MCP-style tool integrations.
